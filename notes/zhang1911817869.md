**三、核心概念笔记**

| 概念 | 解释 |
| --- | --- |
| 钱包 | 链上账户工具，比如 MetaMask。它不是“存钱软件”，更像你的链上身份入口。 |
| 私钥 | 钱包控制权。谁有私钥，谁就能控制资产。不能发给任何人。 |
| 公钥/钱包地址 | 可以公开，用于收款、查询链上记录。 |
| 测试网 | 用来练习和开发的链，测试币没有真实价值。 |
| 水龙头 | 免费领取测试币的网站。通常每天只能领一次。 |
| 测试币 | 测试网上的 Gas 费燃料，不能当真钱。 |
| 交易哈希 | 每笔链上交易的唯一编号。可以在区块浏览器查询。 |
| 区块浏览器 | 查询地址、交易、合约、Gas、时间戳等链上信息的工具。 |
| Gas 费 | 链上操作的手续费，公式大致是 Gas 消耗量 × Gas 单价。 |
| 智能合约 | 部署在区块链上的程序，满足条件后自动执行，部署后通常不可随意修改。 |
| Remix | 在线 Solidity 合约编译和部署工具。 |

**四、Gas 费怎么理解**  
Gas 就是链上操作的“燃料费”。

简单理解：

```
Gas 费 = 操作复杂度 × 当前油价
```

不同操作费用不同：

| 操作 | Gas 消耗 |
| --- | --- |
| 普通转账 | 较低 |
| 部署合约 | 较高 |
| 调用写入函数，比如留言 | 会消耗 Gas |
| 查询只读函数 | 通常不消耗 Gas 或不需要发交易 |

留言字数越多，理论上链上存储数据越多，Gas 可能越高。测试网费用很小，但如果放到真实公链上，成本可能明显增加。

**五、智能合约留言板的意义**  
留言板合约虽然简单，但它覆盖了智能合约最基础的几个动作：

| 动作 | 代表能力 |
| --- | --- |
| 写留言 | 向链上写入数据 |
| 查留言 | 读取链上数据 |
| 记录钱包地址 | 识别是谁操作的 |
| 记录时间戳 | 记录什么时候操作 |
| 触发事件 | 让前端或区块浏览器能追踪交互 |
| 防止空留言 | 合约里的基础校验逻辑 |

所以它不是为了“做一个真正有用的留言板”，而是让你理解：  
**智能合约就是一段放在链上的程序，每次写入都会留下不可篡改的记录。**

**六、新手最容易错的地方**

1.  把私钥发给别人。  
    这是最严重错误，等于把钱包送人。
    
2.  以为测试币有价值。  
    测试币只用于练习和测试，不能兑换真钱。
    
3.  在区块浏览器里找交易按钮。  
    区块浏览器主要是查询工具，不是交易所。
    
4.  钱包网络选错。  
    如果你在 Monad / Mod 测试网操作，钱包也必须切到对应测试网。
    
5.  没有测试币就想部署合约。  
    部署合约、留言、转账都需要 Gas，所以必须先领水。
    
6.  以为合约部署后可以随便改。  
    链上合约部署后通常不可篡改，所以正式部署前要先测试。
